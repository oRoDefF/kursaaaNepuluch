ИНСТРУКЦИЯ ПО ЗАПУСКУ ИНТЕРНЕТ-МАГАЗИНА
1. ПРЕДВАРИТЕЛЬНАЯ ПОДГОТОВКА
1.1. Установка PostgreSQL:
   - Скачайте и установите PostgreSQL с официального сайта
   - Запомните пароль для пользователя postgres
   - Убедитесь, что сервер PostgreSQL запущен
1.2. Установка Python:
   - Убедитесь, что установлен Python 3.8 или выше
   - Проверьте версию: python3 --version
2. НАСТРОЙКА ПРОЕКТА
2.1. Активация виртуального окружения:
   ```bash
   source venv/bin/activate
   ```
2.2. Установка зависимостей:
   ```bash
   pip install -r requirements.txt
   ```
2.3. Настройка базы данных PostgreSQL:
   Войдите в PostgreSQL:
   ```bash
   psql -U postgres
   ```
   Создайте базу данных:
   ```sql
   CREATE DATABASE electronics_store_db;
   \q
   ```
2.4. Применение миграций:
   ```bash
   python manage.py migrate
   ```
2.5. Создание суперпользователя:
   ```bash
   python manage.py createsuperuser
   ```
   Введите логин, email и пароль для администратора
2.6. Заполнение тестовыми данными:
   ```bash
   python manage.py populate_db
   ```
3. ЗАПУСК СЕРВЕРА
3.1. Запуск сервера разработки:
   ```bash
   python manage.py runserver
   ```
3.2. Проверка работы:
   - Откройте браузер
   - Перейдите по адресу: http://localhost:8000/
   - Должна открыться главная страница магазина
4. ДОСТУП К АДМИНИСТРАТИВНОЙ ПАНЕЛИ
4.1. Вход в админ-панель:
   - Перейдите по адресу: http://localhost:8000/admin/
   - Введите логин и пароль суперпользователя
   - Вы получите доступ к управлению:
     * Товарами
     * Категориями
     * Заказами
     * Корзинами
     * Пользователями
5. ТЕСТИРОВАНИЕ ФУНКЦИОНАЛА
5.1. Тестирование для пользователей:
   - Просмотр товаров по категориям
   - Добавление товаров в корзину
   - Оформление заказа
   - Просмотр истории заказов
5.2. Тестирование для администраторов:
   - Добавление новых товаров
   - Редактирование существующих товаров
   - Управление заказами
   - Изменение статусов заказов
6. ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ
6.1. Ошибка подключения к PostgreSQL:
   Проблема: "connection to server at localhost failed"
   Решение:
   - Убедитесь, что PostgreSQL запущен
   - Проверьте настройки в settings.py
   - Проверьте пароль пользователя postgres
6.2. Ошибка миграций:
   Проблема: "No migrations to apply"
   Решение:
   ```bash
   python manage.py makemigrations
   python manage.py migrate
   ```
6.3. Ошибка статических файлов:
   Проблема: "Static files not found"
   Решение:
   ```bash
   python manage.py collectstatic
   ```
6.4. Ошибка медиа файлов:
   Проблема: "Media files not loading"
   Решение:
   - Убедитесь, что папка media существует
   - Проверьте права доступа к папке
7. КОМАНДЫ ДЛЯ РАЗРАБОТКИ
7.1. Создание новых миграций:
   ```bash
   python manage.py makemigrations
   ```
7.2. Применение миграций:
   ```bash
   python manage.py migrate
   ```
7.3. Создание суперпользователя:
   ```bash
   python manage.py createsuperuser
   ```
7.4. Запуск shell Django:
   ```bash
   python manage.py shell
   ```
7.5. Проверка проекта:
   ```bash
   python manage.py check
   ```
